# Laboratory Work №3: Class Diagrams

## Overview
Folder contains Class Diagrams. These diagrams represent the static structure of the systems, showing classes, their attributes, methods and the relationships between objects.

## Contents
The following diagrams are included in this folder:

1. **ATM System (`ATM system case diagram.jpg`)**
   - **Classes:** Bank,ATM,Debit,SavingsAccount,Transaction.
   - **Key Features:** Shows inheritance (SavingsAccount is a type of Debit), associations between Bank and ATM, and transaction logging.

2. **Car Insurance System (`car insurance case diagram.jpg`)**
   - **Classes:** Insurance Company,Customer,Policy,Vehicle,Claim.
   - **Key Features:** Shows the relationships between policyholders, their vehicles, and the claims filed within the insurance system.

3. **Medical Clinic System (`Clinic system case diagram.jpg`)**
   - **Classes:** Person,Patient,Doctor,MedicalRecord,Appointment.
   - **Key Features:** Shows a clear hierarchy where Patient and Doctor inherit from Person, and the composition relationship between Patient and MedicalRecord.

## Objective
The goal of this lab was to:
- Identify system classes and their properties.
- Define data types for attributes (INT, STRING, DATE, etc.).
- Model object-oriented relationships like Inheritance, Association, and Composition.
