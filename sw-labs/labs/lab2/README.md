# Laboratory Work №2: Use Case Diagrams

## Overview
Folder contains Use Case diagrams. These diagrams illustrate the functional requirements and interactions between actors and the systems for various scenarios.

## Contents
The following diagrams are included

1. **ATM System (`ATM use case.jpg`)**
   - **Actors:** User,Bank.
   - **Functionality:** Shows core banking operations such as withdrawing cash, checking balance, depositing money, and changing PIN, including the mandatory authentication process.

2. **Car Insurance System (`car ins.jpg`)**
   - **Actors:** Customer,Repair Shop,Bank.
   - **Functionality:** Describes the claim process, policy verification, damage assessment, and the decision logic for approving or rejecting a claim.

3. **Medical Clinic System (`Medical clinic.jpg`)**
   - **Actors:** Patient,Receptionist,Doctor,Bank.
   - **Functionality:** Illustrates clinic workflows including appointment booking, cancellations, medical history viewing, and payment processing.

## Objective
The goal of this lab was to:
- Identify system actors and their goals.
- Define system boundaries.
- Model relationships using `<<include>>` and `<<extend>>` associations.
